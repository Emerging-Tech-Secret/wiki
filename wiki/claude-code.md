# Claude Code

Claude Code is described by Liu et al. as an agentic coding tool that can run shell commands, edit files, and call external services on behalf of a user [ref_2].
The paper analyzes Claude Code through publicly available TypeScript source code and frames the system as a production coding-agent architecture rather than an autocomplete-style assistant [ref_2].

## Design values and principles

The paper identifies five values motivating Claude Code's architecture: human decision authority, safety and security, reliable execution, capability amplification, and contextual adaptability [ref_2].
The paper maps those values to thirteen design principles, including deny-first human escalation, graduated trust, defense in depth, externalized programmable policy, progressive context management, append-only durable state, isolated subagent boundaries, and graceful recovery [ref_2].

## Runtime loop

Claude Code's central runtime is described as a reactive loop that assembles context, calls the model, routes tool-use blocks through permission checks, executes approved tools, appends tool results, and repeats until the model produces no further tool-use blocks [ref_2].
The paper explicitly relates this loop to the ReAct pattern, where reasoning and tool invocations are generated by the model and environmental results feed the next iteration [ref_2][ref_1].

## Permission and safety model

The paper describes seven permission modes in Claude Code: plan, default, acceptEdits, auto, dontAsk, bypassPermissions, and bubble [ref_2].
The permission system evaluates deny rules before allow rules, so broad deny rules cannot be overridden by narrower allow rules [ref_2].
The auto mode uses an ML-based classifier for requests that do not pass fast-path checks when its feature flag is active [ref_2].

## Context management and persistence

Claude Code uses a five-layer context-management pipeline consisting of budget reduction, snip, microcompact, context collapse, and auto-compact [ref_2].
The paper describes Claude Code's session transcripts as mostly append-only JSONL files that record conversation messages, compaction markers, file-history snapshots, attribution snapshots, and content-replacement records [ref_2].
The paper states that resume and fork rebuild conversations from transcripts but do not restore session-scoped permissions [ref_2].

## Extensibility and subagents

Claude Code exposes four extension mechanisms: MCP servers, plugins, skills, and hooks [ref_2].
The Agent tool can spawn isolated subagents whose definitions can specify prompts, tools, disallowed tools, models, effort, permission modes, MCP servers, hooks, maximum turns, skills, memory scope, background behavior, and isolation mode [ref_2].
The paper distinguishes AgentTool from SkillTool by saying that AgentTool spawns a new isolated context while SkillTool injects instructions into the current context window [ref_2].
